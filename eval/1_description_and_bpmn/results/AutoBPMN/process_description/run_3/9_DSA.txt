The process begins with a startevent. Following this, an "Application for Trusted Flagger Status" is submitted. This application is then reviewed by the "Digital Services Coordinator Reviews Application".

At this point, a decision is made. If the application "Meets Conditions", the "Trusted Flagger Status" is awarded. If the conditions are not met, the application is rejected, and the process ends.

After the status is awarded, there's a check. If everything is in order, the "Trusted Flagger Publishes Annual Report". This report is then sent by the "Trusted Flagger" to the "DSC" (Digital Services Coordinator). Subsequently, the "Trusted Flagger Makes Report Publicly Available". The "DSC" then "Communicates Trusted Flagger Info to Commission and Board".

Following this communication, there's a branching point. One path leads to the "Commission Issues Guidelines if Necessary". Another path leads to the "Commission Publishes Trusted Flagger Database".

The process then checks for reported issues. If "No Issues Reported", the "Trusted Flagger Continues Operations" and proceeds to publish their next annual report. If "Provider Reports Insufficiently Precise Notices", the "DSC Suspends Status for Investigation". The "DSC Investigates Notices".

During the investigation, a decision is made:
* If "Conditions No Longer Met", the "DSC Revokes Status". The "DSC Communicates Revocation to Commission and Board".
* If "Conditions Still Met", the "DSC Reinstates Status". The "DSC Communicates Reinstatement to Commission and Board".

Both the revocation and reinstatement communication paths eventually lead back to a point where the "Commission Publishes Trusted Flagger Database".

There are also several loops and checks related to the "Commission Publishes Trusted Flagger Database" and the "Commission Issues Guidelines if Necessary" steps, which can lead back to the "Trusted Flagger Continues Operations" or trigger an escalation.

The process also includes a path where if "No Issues Reported", the "Commission Publishes Trusted Flagger Database". This can also lead to the "Trusted Flagger Continues Operations" and the subsequent annual report publication.

The process includes multiple escalation points, indicated by an upward-pointing arrow, which can lead back to various decision points or the end of the process.

The process concludes with an endevent.
